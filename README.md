# FedXAI — Federated & Explainable AI for Chronic Disease Prediction

Master's thesis project implementing privacy-preserving disease prediction
using Federated Learning, Multi-class Classification, and Explainable AI.

## System Architecture

```
3 Hospital Nodes (simulated locally)
    ↓  model weights only (never raw data)
FL Aggregation Server (FedAvg)
    ↓  global model
Prediction Engine (multi-class: Diabetes, CVD, Hypertension, Healthy)
    ↓  predictions
XAI Engine (SHAP + Groq LLM)
    ↓  explanations + clinical reports
Streamlit Dashboard
```

## Tech Stack (all free)

| Component | Tool |
|---|---|
| Federated Learning | `flwr` (Flower) |
| ML Model | `scikit-learn` LogisticRegression |
| Differential Privacy | `diffprivlib` (IBM) |
| Explainability | `SHAP` |
| LLM Reports | Groq API → `llama-3.1-8b-instant` |
| Dashboard | Streamlit + Plotly |
| Dataset | CDC BRFSS 2015 (Kaggle, free) |

## Setup

### 1. Clone and install
```bash
git clone <your-repo>
cd fedxai
pip install -r requirements.txt
```

### 2. Configure environment
```bash
cp .env.example .env
# Edit .env and add your Groq API key (free at https://console.groq.com)
```

### 3. Get the dataset
1. Go to: https://www.kaggle.com/datasets/alexteboul/diabetes-health-indicators-dataset
2. Download `diabetes_binary_health_indicators_BRFSS2015.csv`
3. Save it as `data/raw_data.csv`

### 4. Prepare data
```bash
python data/prepare_data.py
```
This splits the dataset into 3 hospital partitions (A, B, C) — simulating data silos.

### 5. Run federated learning simulation
```bash
# Basic run (5 rounds, no DP)
python run_simulation.py

# With differential privacy
python run_simulation.py --rounds 10 --dp --dp-epsilon 0.5

# More rounds for better accuracy
python run_simulation.py --rounds 20
```

### 6. Generate XAI explanation
```bash
python xai/explain.py --patient-id 0 --hospital A
```

### 7. Launch dashboard
```bash
streamlit run dashboard/app.py
```

## How Federated Learning Works Here

```
Round 1:
  Server → sends initial weights to Hospital A, B, C
  Hospital A: trains on its 19,500 patients → sends back weight updates
  Hospital B: trains on its 19,500 patients → sends back weight updates
  Hospital C: trains on its 19,500 patients → sends back weight updates
  Server: FedAvg aggregation → new global weights

Round 2..N:
  Repeat — each round the global model improves without any patient data moving
```

**Privacy guarantee:** Raw patient records NEVER leave the hospital.
Only floating-point model weight arrays are transmitted.
